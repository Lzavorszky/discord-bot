### POLYMICROBIAL_SPECTRUM_RULE

Core rule:
- In polymicrobial infections, select therapy corresponding to the highest antimicrobial spectrum tier required by the detected organisms and resistance markers.
- Therapy must cover all detected clinically relevant organisms.
- If requirements conflict, recommend infectious diseases consultation.

Spectrum hierarchy:
- Tier 1: ceftriaxone
- Tier 2: cefepime
- Tier 3: ertapenem
- Tier 4: meropenem + colistin

Examples:
- Streptococcus pneumoniae + Haemophilus influenzae -> Tier 1 -> ceftriaxone
- Streptococcus pneumoniae + Pseudomonas aeruginosa -> Tier 2 -> cefepime
- Escherichia coli + CTX-M -> Tier 3 -> ertapenem
- Acinetobacter calcoaceticus-baumannii complex -> Tier 4 -> meropenem + colistin
- Pseudomonas aeruginosa + CTX-M positive Enterobacterales -> conflicting broad-spectrum requirements -> ID consultation

## INFO_BLOCKS

### recognition

BioFire interpretation only answers when at least one pathogen is supplied.
Resistance markers are interpreted only together with the detected pathogen list.
Pathogen names must be treated as microbiology entities.
Streptococcus pneumoniae, Strep pneumoniae, Strep pneumo and S. pneumoniae are organism aliases for this protocol.
Do not route these to CAP because of the pneumoniae substring.

### polymicrobial_spectrum

In polymicrobial infections, select the highest required antimicrobial spectrum tier among detected organisms and resistance markers.
The selected therapy must cover all clinically relevant detected organisms.
If the combination creates conflicting broad-spectrum requirements, recommend infectious diseases consultation.

### resistance_genes

Resistance markers are BioFire-positive only in the context of detected pathogen(s), so they must not be interpreted alone.
CTX-M suggests ESBL-producing Enterobacterales and upgrades relevant detected Enterobacterales to ertapenem.
Carbapenemase genes are urgent ID consultation triggers and point to Tier 4 handling when detected with a pathogen.
mecA/C or MREJ with Staphylococcus aureus suggests MRSA and points to vancomycin.
Do not apply CTX-M to Streptococcus pneumoniae or other non-Enterobacterales.

### limitations

Routine use in ventilator-associated pneumonia is generally discouraged because many nosocomial pathogens are absent from the panel, resistant ICU flora are incompletely represented, Stenotrophomonas is not included, positive respiratory PCR may represent colonization, and prior microbiology data is often already available.
The panel may still be used for targeted diagnostic questions.

## RESTRICTED_OUTPUTS

Never provide:
- antimicrobial doses from BioFire
- renal adjustment from BioFire
- organism-specific interpretation when no pathogen is supplied
- standalone resistance-marker interpretation without a detected pathogen
- CAP syndrome pathway selection from this protocol
- joint infection panel interpretation from this pneumonia panel protocol
- CTX-M escalation for non-Enterobacterales alone
- narrower therapy in polymicrobial infection when a broader tier is required

For dosing requests:
- use LINKS
- if the linked target protocol is missing, say the target_missing_behavior sentence

For info requests:
- answer only from INFO_BLOCKS
- do not add antimicrobial dosing

## SAFETY_RULES

- At least one pathogen is required before interpretation.
- Resistance markers are interpreted only with detected pathogen(s); if marker-only input is supplied, ask for the pathogen.
- No multi-turn question tree is needed; if missing, return DEFAULT_ANSWER only.
- Keep last_selected_item so a follow-up dosing request can activate LINKS.
- If multiple antimicrobials are selected and the user asks for dose, ask which antimicrobial unless the user named one.
- If GFR/renal data is supplied, store it and transfer it through LINKS; do not use it inside BioFire.
